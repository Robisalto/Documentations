Concepts réseaux

Généralités

 

  • Le réseau permet de connecter des équipements entre eux pour faire transiter de l'information
     
  • On s'appuie sur le modèle OSI pour découper en 7 couches les différentes logiques / fonctionnalités nécessaires
     
  • Un modèle simplifié en 5 couches : TCP/IP
     
  • Les couches basses sont orientées communication / materiel
     
  • Les couches hautes sont orientées données / transports et s'appuient sur les couches basses

TCP vs UDP

 

  • Deux protocoles de transport
  • Utilisent des ports pour la connexion aux services
  • TCP est un protocole qui fonctionne en mode connecté, utilisé la ou la fiabilité est privilégiés
  • UDP fonctionne en mode non connecté, utilisé la ou la faible latence est prioritaire comme le temps réel

L'adresse MAC

 

  • L'équipement, au travers d'une interface, est connecté au réseau
    => Une interface spéciale : lo qui correspond à l'interface de loopback
  • L'interface possède une adresse physique, sur 8 octets : l'adresse MAC
  • Cette adresse doit être unique sur le réseau et est utilisée pour la communication entre éléments d'un même réseau
  • Un moyen de voir son adresse MAC sous linux : ip addr show <nom_interface>
$ ip addr show wlp59s0 
2: wlp59s0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc noqueue state UP group default qlen 1000
    link/ether 24:41:8c:64:4b:00 brd ff:ff:ff:ff:ff:ff
    inet 192.168.1.125/24 brd 192.168.1.255 scope global dynamic noprefixroute wlp59s0
       valid_lft 39134sec preferred_lft 39134sec

L'adresse IP

 

  • Pour la communication entre réseaux, on affecte une adresse IP
  • Cette IP représente une ressource avec laquelle on dialoguer
  • Une adresse IP spéciale : 127.0.0.1 qui est l'adresse locale ou de loopback
    => en fait toutes les adresses en 127.x.x.x
  • L'adresse IP fait partie d'un réseau IP, la notation complète est adresse de <adresse_ip>/<CIDR>
    Ex: 192.168.0.12/24
  • Deux versions du protocole IP :
    • v4 : sur 32 bits
    • v6 : sur 128 bits

Un réseau IP

 

  • Un ensemble d'adresses IPs
  • Initialement découpé en classes IP
  • Peut être découpé en sous réseaux
  • Le nombre d'adresses disponibles dépend de la taille du réseau (la valeur du CIDR)
  • Des adresses sont réservées :
    • La première adresse est l'adresse du réseau
    • La dernière adresse est l'adresse de broadcast

 

Un sous-réseau IP

 

  • Un bloc compris dans un réseau plus conséquent
  • On découpe la plage définie initialement pour segmenter en de plus petits réseaux
  • Exemple : un réseau en /24 (max 254 hosts) découpé en réseaux /28 (max 14 hosts)
  • Un host en /24 peut communiquer avec ses sous réseaux si présents sur le même réseau physique
  • Les sous réseaux ne peuvent pas se parler entre eux sans routage
  • Une calculatrice pour calculer les sous réseaux possibles et leurs caractéristiques

 

Le routage

 

  • Lorsque l'on veut faire communiquer des réseaux différents
  • Indication pour connaitre le chemin à emprunter
  • La lsite des chemins est la table de routage
  • Un chemin est une route
  • Une route spéciale, la route par défaut : 0.0.0.0
    => gère le accès vers tout ce qui n'est pas défini par des routes explicites
  • Pour lister les routes : route
  • Pour ajouter des routes : ip route
  • Pour passer d'un réseau à l'autre, on passe par des passerelles (gateway)

ARP

 

ARP est un protocole qui permet de connaitre l'adresse Mac de l'interface associée à une IP

  1. Une requête est envoyée en broadcast pour connaitre le propriétaire d'une IP
  2. Réponse du proprétaire de l'IP uniquement directement à l'emeteur avec l'adresse MAC
     

Commandes Linux pour visualiser les correspondances connues: arp -a

DNS

 

  • Système de gestion de nom hierarchique et distribué
  • Protocole réalisant de la résolution de noms en adresse IP
  • Plusieurs types d'enregistrements DNS
    • A : nom -> adresse IPv4
    • AAAA : nom -> adresse IPv6
    • CNAME : alias DNS -> nom DNS
    • PTR : adresse IP -> nom DNS
    • TXT : nom DNS -> contenu textuel
    • MX : serveurs en charge de la réception des mails

Bonding

 

  • Aggrégation d'interfaces réseaux
  • Est ensuite utilisable comme une interface classique

 

 

  • Permet :
    • De combiner les capacités des interfaces
    • De disposer d'une tolérance à la panne

 

VIP

 

  • Adresse IP virtuelle - Virtual IP
     
  • Utilisé en frontal de services
     
  • Souvent utilisé pour accéder à un service actif / passif
     
  • Peut être basculé d'une instance à une autre

Firewall

 

  • Utilisé pour filtrer le trafic réseau (n4) ou applicatif (n7)
  • Permet d'autoriser ou de refuser en fonction :
    • d'adresse sources ou destination
    • du sens du trafic (entrant, sortant)
    • des ports source ou de destination
    • des protocoles
    • ...
  • Peut être utilisé pour du suivi de trafic

 

DMZ

 

  • Zones réseaux utilisées pour cloisonner les services et limiter les accès et flux à ouvrir
     
  • Sert de zone d'exposition pour les services devant être accéder depuis des zones non sures
     
  • Evite d'exposer directement le réseau interne
     
  • Limite le trafic à gérer
     
  • Plus sécurisé

Quelques commandes utiles

Réseau

  • ping : teste la disponibilité d'un équipement en utilisant ICMP
  • traceroute : identifie les éléments entre la source et une destination
  • arp : gestion de la résolution ARP, -a pour lister les éléments connus
  • netstat : affiche les connexion et ports en écoute
  • ip : gestion des interfaces et des routes
  • route : affiche la table de routage

 

DNS

  • dig : résolution de noms
  • nslookup : interrogation de serveurs DNS directement

 

Services

  • SSH : connection sécurisée entre une source et une destination
  • SCP : transfert de données via SSH
  • curl : outils pour consommer desservices (généralement web)
  • ss : outil de diagnostique
  • UFW : Firewall sur Linux
